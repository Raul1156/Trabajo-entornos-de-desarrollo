Comandos de Git
Git Push
Analogía: Has hecho algunas mejoras en tu copia del libro y ahora quieres que los demás también tengan esas mejoras. Entonces, "envías" tus cambios a la biblioteca principal para que todos los puedan ver. Explicación: git push envía tus commits (cambios) al repositorio remoto, compartiéndolos con otros colaboradores que tengan acceso al repositorio.
Git Commit
Imagina que cada vez que escribes o editas un párrafo o capítulo de un libro, decides hacer una copia de esa versión. Hacer un commit es como tomar una "foto" de ese estado del libro. Esta foto no se borra y puedes volver a ella más adelante. Escribir una página nueva y haces un commit, dejando registrada esa versión. Luego, escribes más y haces otro commit. Si algo sale mal o no te gusta después, puedes volver a cualquiera de las versiones anteriores.
Git Checkout
Cuando quieres revisar cómo estaba el libro en una versión específica, haces "checkout" a ese momento o a otra rama. Es como ir a una sección diferente de la biblioteca donde guardas las versiones y seleccionar la que quieres ver o en la que quieres trabajar. Si estás en la versión actual del libro y decides ver cómo estaba en la versión de hace dos semanas, haces checkout a ese punto en el tiempo y trabajas desde ahí. O si quieres trabajar en la "rama alternativa" del libro, haces checkout a esa rama para modificar solo esa parte.
Git Branch
Piensa en un branch como una "rama" del libro. Empiezas con la historia principal, pero decides explorar una versión alternativa de un capítulo sin afectar la trama original. Abres una "rama nueva" (branch) y haces cambios ahí. En tu libro, creas una rama llamada "nueva-trama" y trabajas en una historia alternativa. Al final, puedes decidir si te gusta esa versión y la unes a la historia principal o si simplemente la dejas como un experimento.
Git Status
Status es como revisar tu escritorio para ver qué páginas has cambiado, cuáles están listas para guardar, y cuáles aún están pendientes. git status te muestra el estado actual de los archivos en tu repositorio. Te indica qué archivos han cambiado, cuáles están listos para commit, y cuáles aún no se han añadido.
Git Clone
Imagina que te dan una copia de un libro de otro autor y quieres tener tu propio ejemplar para leer y hacer anotaciones. "Clonar" es hacer una copia completa del libro en tu propia biblioteca. git clone toma un repositorio completo y lo copia a tu ordenador. Esto incluye todo el historial de cambios, ramas, y archivos del proyecto.
